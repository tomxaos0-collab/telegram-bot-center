# Telegram Bot Center — Каркас Десктопного Приложения

Рабочий каркас десктопного приложения для управления Telegram-ботами, вебхуками и рассылками. Интерфейс выполнен в премиальном минималистичном темном стиле (по аналогии с Linear, Raycast и Vercel) на русском языке.

Стек технологий: **React 19 + TypeScript + Tauri v2 + Tailwind CSS4**.

---

## 🚀 Быстрый старт при веб-разработке

Если вы хотите запустить и протестировать веб-интерфейс в браузере (на порту `3000`):

1. Установите зависимости Node.js:
   ```bash
   npm install
   ```
2. Запустите локальный веб-сервер:
   ```bash
   npm run dev
   ```
3. Откройте в браузере адрес [http://localhost:3000](http://localhost:3000).

## 🛠 Настройка для сборки Десктопного Приложения в Linux (Arch Linux / KachiOS)

Для компиляции в нативный бинарный файл под Linux (через Rust-ядро Tauri) вашей системе потребуются системные библиотеки и компиляторы GCC/LLVM.

### 1. Установка системных зависимостей (Arch / KachiOS / Manjaro)

Выполните установку необходимых пакетов компилятора, GTK-библиотек и веб-движка WebKit из системных репозиториев Pacman:

```bash
sudo pacman -Syu --needed \
  base-devel \
  curl \
  wget \
  file \
  openssl \
  webkit2gtk-4.1 \
  libappindicator-gtk3 \
  librsvg \
  glib2 \
  pango \
  harfbuzz \
  gtk3
```

> **Примечание:** Tauri v2 использует WebKit2GTK версии `4.1` (вместо `4.0` в Tauri v1). Убедитесь, что установлен пакет `webkit2gtk-4.1`.

### 2. Установка компилятора Rust (Rustup)

Tauri требует стабильную версию Rust для сборки бэкенда. Установите Rustup:

```bash
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
```

Перезапустите терминал или выполните:
```bash
source "$HOME/.cargo/env"
```

## 💻 Команды запуска и сборки Tauri

После установки системных зависимостей и Node-пакетов вы можете использовать следующие CLI-скрипты:

* **Инструмент запуска разработчика (Tauri Dev-Mode):**
  Запускает локальный веб-сервер и открывает интерфейс в нативном десктопном окне с включенным Hot-Reload.
  ```bash
  npm run tauri dev
  ```

* **Сборка готового дистрибутива (Tauri Build-Mode):**
  Компилирует оптимизированный бинарный исполняемый файл и создает упакованные установщики (например, `.AppImage` или `.deb` пакеты) в папке `src-tauri/target/release/bundle/`.
  ```bash
  npm run tauri build
  ```

## 📂 Структура проекта

* `src-tauri/` — Ядро нативного десктопного приложения (Rust).
  * `tauri.conf.json` — Основной конфигурационный файл Tauri (параметры окна, разрешения, пути дистрибутива).
  * `Cargo.toml` — Зависимости Rust crates.
  * `src/main.rs` — Точка входа в Rust бинарник.
  * `src/lib.rs` — Библиотечная логика инициализации Tauri-приложения.
  * `capabilities/default.json` — Профиль прав безопасности Tauri v2.
* `src/` — Пользовательский интерфейс и фронтенд (React + TS + Tailwind).
  * `App.tsx` — Центральный менеджер состояния (симуляция входящих обновлений, роутинг экранов).
  * `components/` — Модульные реактивные компоненты экранов:
    * `Sidebar.tsx` — Панель навигации по разделам.
    * `Header.tsx` — Верхняя служебная панель с эмуляцией заголовков ОС Arch Linux.
    * `BotManager.tsx` — Экран управления ботами (форма добавления, блокировки и статусы).
    * `ChatManager.tsx` — Живой чат с симулированной цепочкой ответов пользователей.
    * `MessageSender.tsx` — Мощный конструктор рассылок с визуальным превью на смартфоне.
    * `WebhookTester.tsx` — Конструктор тестовых вебхук-запросов и монитор POST.
    * `LogView.tsx` — Эмулятор системного терминала (интерактивный вывод потока логов ядра).
    * `SettingsView.tsx` — Конфигурация параметров SQLite и тем оформления.
  * `types.ts` — Глобальные интерфейсы и типы данных.
  * `index.css` — Подключение Tailwind и шрифтов Inter / JetBrains Mono.
